# Regras do Projeto Gerentask

- **Atualização Contínua de Documentação:** Toda vez que você (agente) fizer uma alteração na aplicação (adicionar uma nova funcionalidade, alterar a estrutura do banco de dados, modificar permissões, etc), você DEVE obrigatoriamente atualizar os arquivos de documentação do projeto, especialmente:
  - `docs/requisitos.md`: Refletindo novos casos de uso e requisitos implementados.
  - `docs/arquitetura.md`: Refletindo alterações na modelagem de dados e tecnologias usadas.
  - `README.md` (se aplicável, para comandos ou bibliotecas).
  
Nunca encerre o seu turno de desenvolvimento sem garantir que as documentações do projeto estão sincronizadas com o estado atual do código.
